## 4. Application development history (Phases 1–2, unchanged historical record)

This section is preserved as written at the end of Phase 2 — it describes what was true then, not necessarily what is current now. See §6 for the current implementation state and §7 for current test/build numbers.

**Phase 1 (scaffold):**
- React + Vite + TypeScript + React Router, plain CSS — no backend, no database, no auth, no state-management library.
- Routes implemented: `/`, `/chapter/1`, `/chapter/1/topic/:topicId`, plus a catch-all not-found route.
- Bilingual shell: `LanguageContext` (session-local React state), `dir`/`lang` synced on the document root on toggle.
- Persistent, non-dismissible internal-status banner present on every route.
- Local dev server, TypeScript build, and browser verification (desktop/tablet/mobile, keyboard focus, unknown-topic handling) were completed and reported at the end of that phase.

**Phase 2 (real content rendering):**
- **Content loading:** `apps/chapter1-mvp/src/content/rawImports.ts` is the single file that imports the four canonical JSON files and four SVGs directly from `docs/content-design/chapter-01/pilot/` — no duplication anywhere (still true — re-verify by confirming no `.svg` or `ch01-t*-content.json` files exist inside `apps/chapter1-mvp/src` or `public`). `vite.config.ts`'s `server.fs.allow` is scoped narrowly to `docs/content-design/chapter-01/` only.
- **Read-only adapter:** `src/content/adapter.ts` transforms validated raw records into a normalized UI model (`src/types/normalized.ts`) and never mutates or writes back to source data.
- **Schema:** `src/types/pilotSchema.ts` models the actual schema, `schemaVersion: "2.3.0"` across all four topics, including two documented real irregularities (file-level `topicTitle`/`topicTitleAr` are plain strings, not the `LocalizedText` shape used elsewhere; `instructorScript` has no `visibility` field).
- **Validation:** `src/content/validate.ts` — hand-written TypeScript type guards (no Zod), producing `AdapterDiagnostic[]` for every failure mode; never invents fallback content, never silently drops an invalid record.
- **Equations:** `src/content/equationRenderer.tsx` — semantic HTML with true `<sup>`/`<sub>`, conservative per-topic italic whitelists, LTR-isolated runs inside Arabic prose. No KaTeX/MathJax (still true — see §6, unchanged).
- A real regression was found and fixed during Phase 2 verification: applying the italic-variable whitelist to prose text caused t08's `"a"` (acceleration variable) to also italicize the English article "a". Fixed by splitting into a full whitelist (equation-dense/symbolic fields) and a prose-safe whitelist (excludes ambiguous English words); locked in with a regression test that still passes today.
- No canonical content is duplicated into React components — components receive normalized data as props only. No runtime machine translation anywhere. No synthesized summaries, key concepts, or learning objectives.

**Phase 3 (internal QA and accessibility hardening) and Phase 4 (project-owner review) — summarized, full detail in their own reports:**
- Phase 3 implemented `reviewQuestion` rendering (see §6), rebuilt `VisualViewer`'s enlarge dialog on the native `<dialog>` element with a verified focus trap/background-inertness/Escape/scroll-lock, ran a measured WCAG contrast audit (fixed two real failures), completed a full bilingual/responsive QA pass at 375/768/1280/1600px, and added 20 new automated tests.
